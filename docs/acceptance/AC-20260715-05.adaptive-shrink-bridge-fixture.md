## 実証済み

- Playwrightのtimeout失敗列を、新規ブラウザでstrict replayし、同一の実行statusを保ったまま不要な非変更操作を除去した。
- destructive action、入力、外部送信などを含み得る候補は縮約対象から除外した。
- Airtest/PocoはLakdaから外部processを起動せず、operator管理のloopback JSON bridgeのcapability handshakeと共通SPIで実行した。
- 外部AdapterはendpointとinitialTargetが未指定なら設定検証でfail-closedに拒否した。
- 同一runのgraph、trace、coverage、oracle、shrink reportは既存HATE/v1 manifestから発見できる。
- Security adapterはsecurityProfileRefと認可recordを必須とし、scope・有効期間・rate・production制約をcandidateごとに評価する。deny時はbridgeを実行せずcandidate-deniedを保存する。
- active Security mutationはoperator管理のsecurity-controlとcleanup endpointを必須とし、control/cleanupが利用不能ならfail-closedにする。
- raceは通常workerの並列化ではなく専用schedulerで実行する。schedulerはconcurrencyを守り、kill switch後にqueued participantを開始せず、開始済みparticipantの終了後にcleanupを記録する。
- bridge実行例外も構造化したinfrastructure errorとして保存し、cleanupを必ず試行する。
- ZAP alertは外部Security bridgeが生成する非変更candidateとして取り込み、riskがhighでもsecurity oracleのverdictはcandidateに固定する。confirmedへの昇格には認可済みreplayと明示oracleまたは人手確認recordが別途必要である。
