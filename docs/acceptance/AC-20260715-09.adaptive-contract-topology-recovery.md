# AC-20260715-09: ActionContract・target topology・verified recovery hardening

## Scope

ActionContractのfail-closed評価、操作後の再観測、generic/product/security oracleの分離、graph edgeのoracle参照と制御遷移種別、重み付き探索、Playwrightの主要表示・modal・iframe scope-out・popup・browser event、field単位InputCase、fingerprint検証付きbacktrackを対象とする。

本記録はlocal HTTP fixtureとin-process Chromiumによる実装証跡である。Airtest/Poco実機、認可済みSecurity target、固定16 AC corpus、manual-bb、QEGの本証跡ではない。

## Acceptance Criteria

- ActionContractのenabledWhen、ensures、invariantsを観測値で評価し、未知・欠損・不一致をfail-closedで扱う。
- 各操作後にDOMを再観測し、次のcandidateを更新する。
- generic、product、securityのOracleResultを別々に保持し、graph edgeから証跡参照できる。
- graphでaction、timeout、backtrack、recovery、reset等の遷移種別を区別し、重み付きrandomをseedから決定的に選択する。
- 主要表示要素とDOM modalを操作要素から分離し、JS dialogは別target/eventとして既定拒否する。
- scope外または非HTTP iframeをpartial observationとして扱い、DOMを読まずcandidateを生成しない。
- popup/new tabにopener、trigger action、initial/settled URL、scope、lifecycleを記録する。
- console error、pageerror、crash、request failure、HTTP 5xx、downloadをbounded・redacted eventとして観測する。
- form fieldIdとInputCaseを一致させ、raw入力値をtraceへ保存せずreplayする。
- backtrack後のfingerprintを操作前状態と比較し、不一致ならrecovery-divergenceとして停止する。
- 復旧に成功しても元のtimeout/failure判定を上書きしない。
